## 關鍵技術要點

### 1. 四層記憶模型
```
Layer 3 (全書索引) ← 永遠注入，3000 tokens
Layer 2 (卷摘要)   ← 當前卷+前卷，2500 tokens
Layer 1 (事件)     ← 智能檢索，2500 tokens
Layer 0 (原文)     ← 當前章+前章，10000 tokens
```

### 2. Token 預算分配
- 固定分配：~27,500 tokens
- 彈性空間：根據模型自動調整
- 最保守基準：DeepSeek V3 (46K 可用)

### 3. 自動化流程
- 章節保存 → 自動提取事件
- 卷完結 → 手動生成卷摘要
- 每 20 章 → 重建全書索引
- 生成前 → 一致性檢查

### 4. 實現優先級
1. **Phase 1**（最高 ROI）：事件提取 + 智能上下文
2. **Phase 2**：分卷摘要 + 全書索引
3. **Phase 3**：一致性檢查 + UI 增強
