## 一、各類型代表作品參考庫

## 二、跨類型共同成功模式分析

### A. 開篇黃金法則（前三章）

| 要素 | 說明 | 成功案例 |
|------|------|----------|
| 懸念鉤子 | 第一段就拋出疑問或衝突 | 《詭秘之主》開篇穿越+神秘信件 |
| 主角定位 | 3章內讓讀者知道主角是誰、要什麼 | 《斗破》三年之約，目標明確 |
| 金手指展示 | 暗示或小規模展示主角的特殊能力 | 《大王饒命》負面情緒值系統首次觸發 |
| 第一個小爽點 | 前3章必須有一次小規模的「爽」 | 《凡人》韓立測靈根，意外發現天賦 |
| 世界觀錨點 | 用1-2個細節暗示世界觀的獨特性 | 《遮天》九龍拉棺的震撼開場 |

**核心原則：前三章 = 電梯演講，必須在最短時間內回答「為什麼要繼續讀」。**

### B. 爽點節奏設計

```
章節級爽點（每3-5章一次）
├── 小打臉：言語反擊、小勝一場
├── 技能展示：新能力首次使用
└── 信息差：讀者知道主角底牌，配角不知道

段落級爽點（每10-15章一次）
├── 中型打臉：公開場合碾壓對手
├── 升級突破：境界提升、獲得新裝備
└── 身份揭露：隱藏身份被發現

卷級爽點（每30-50章一次）
├── 大型打臉：擊敗BOSS級反派
├── 地圖升級：進入更高層次的世界
└── 關係突破：重要感情線進展
```

**爽點密度公式：**
- 免費章節（前30章）：每2-3章一個爽點，密度最高
- 付費初期（30-100章）：每3-5章一個爽點，保持節奏
- 中期穩定（100-500章）：每5-8章一個爽點，穿插鋪墊
- 高潮卷：連續爽點轟炸，每章都有推進

### C. 角色設計公式

**主角設計四要素：**
1. 明確的短期目標（活下去/變強/報仇）
2. 深層的長期動機（守護/自由/真相）
3. 性格記憶點（韓立的謹慎、蕭炎的倔強、許七安的毒舌）
4. 成長空間（初始狀態與最終狀態的差距越大越好）

**反派層次設計：**
```
第一層：同輩對手（同學、同門）→ 用於日常打臉
第二層：長輩壓制（家族長老、宗門高層）→ 用於中期衝突
第三層：勢力BOSS（宗主、國王）→ 用於卷級高潮
第四層：幕後黑手（隱藏大佬）→ 用於全書主線
```

**配角功能分類：**
- 忠犬型：無條件支持主角，提供情感溫度
- 解說型：替讀者震驚，烘托主角強大
- 鏡像型：與主角走不同道路，形成對比
- 工具型：提供信息/資源，推動劇情

### D. 伏筆與懸念系統

**三層懸念結構：**
1. 章節懸念（每章結尾）：「他推開門，看到的卻是...」
2. 段落懸念（每個小arc）：「那個神秘人的身份到底是...」
3. 全書懸念（貫穿始終）：「世界的真相究竟是...」

**伏筆埋設原則：**
- 埋設時要自然，不能刻意
- 回收時要有「原來如此」的驚喜感
- 重要伏筆至少暗示3次再揭曉
- 伏筆回收間隔：小伏筆10-30章，大伏筆100-300章

### E. 節奏控制模型

**「三幕四拍」單卷結構：**
```
第一幕：鋪墊（20%）
  → 新地圖/新勢力介紹
  → 新衝突種子埋下
  → 1-2個小爽點維持興趣

第二幕上：發展（30%）
  → 衝突升級
  → 主角遇到困難
  → 中型爽點穿插

第二幕下：轉折（20%）
  → 意外事件打破平衡
  → 主角陷入低谷或面臨抉擇
  → 讀者緊張感最高

第三幕：高潮+收尾（30%）
  → 連續爽點爆發
  → BOSS戰/大型打臉
  → 留下下一卷的鉤子
```

### F. 付費節點設計

**關鍵付費轉化節點（第25-35章）：**
- 必須在付費牆前完成：主角定位、金手指展示、第一個中型爽點
- 付費牆處必須有：強懸念（不看下一章會難受）
- 付費牆後立刻給：爽點回報（讓付費讀者覺得值）

**讀者留存關鍵節點：**
- 第1章：決定是否繼續看
- 第3章：決定是否加入書架
- 第30章：決定是否付費
- 第100章：決定是否追更
- 第300章：決定是否成為鐵粉

## 三、寫作技法規則庫（可直接整合進系統 Prompt）

### 基礎規範（20條，升級版）

**節奏類：**
1. 每章必須有至少一個「推進點」（劇情/角色/世界觀任一維度的推進）
2. 連續鋪墊不超過3章，第4章必須給爽點
3. 戰鬥場景佔比不超過單章40%，避免流水帳
4. 對話與敘述比例建議 4:6 到 5:5
5. 每章結尾必須有鉤子（懸念/反轉/新信息）

**爽點類：**
6. 打臉前必須有足夠的「壓制」鋪墊，壓得越狠爽得越徹底
7. 裝逼要有層次：先小露一手→對方不信→再展示→徹底震驚
8. 升級突破要有儀式感，不能一筆帶過
9. 身份揭露要選在最有戲劇性的時刻
10. 每個爽點後要有「餘韻」——旁人的反應、議論、震驚

**角色類：**
11. 主角每50章要有可感知的成長變化
12. 重要配角出場要有記憶點（外貌/口頭禪/特殊行為）
13. 反派不能純粹是壞，要有自己的邏輯和動機
14. 感情線發展要有阻礙，不能太順利
15. 配角不能搶主角風頭，所有高光時刻歸主角

**世界觀類：**
16. 設定融入劇情中展示，禁止大段設定集式說明
17. 力量體系要有明確的上限和規則，不能隨意打破
18. 新地圖要在3章內讓讀者建立空間感
19. 勢力關係要有灰色地帶，不能非黑即白
20. 每個大設定的揭露都要伴隨劇情衝擊

**文筆類（新增5條）：**
21. 動作描寫要有「鏡頭感」——遠景→中景→特寫
22. 環境描寫服務於氛圍，不超過3句
23. 內心獨白要簡短有力，避免長篇大論的心理分析
24. 對話要有個性區分，不同角色說話風格不同
25. 章節標題要有吸引力，能引發好奇心

## 四、系統整合方案

### 階段一：創作前規劃模組

**功能：小說企劃書生成器**

在開始寫作前，引導作者完成：
```
1. 類型定位 → 根據參考庫推薦該類型的成功模式
2. 核心賣點設計 → 金手指/主角特質/世界觀亮點
3. 爽點路線圖 → 前100章的爽點分佈規劃
4. 角色關係圖 → 主角、女主/男主、反派、配角的關係與功能
5. 分卷大綱 → 每卷的核心衝突與高潮點
```

**AI Prompt 增強：**
根據作者選擇的類型，自動載入該類型的成功模式作為生成參考。

### 階段二：寫作中智能輔助

**功能增強：**

1. **節奏監控器**
   - 追蹤最近N章的爽點密度
   - 當連續3章無爽點時自動提醒
   - 建議下一章應該安排什麼級別的爽點

2. **角色一致性檢查**
   - 比對角色設定卡與實際對話/行為
   - 檢測角色是否「OOC」（Out of Character）
   - 追蹤角色成長弧線是否按計劃推進

3. **伏筆追蹤器**
   - 記錄已埋設的伏筆及預計回收章節
   - 提醒即將到期未回收的伏筆
   - 建議伏筆回收的最佳時機

4. **類型化建議引擎**
   - 根據當前類型的成功模式給出針對性建議
   - 例：玄幻類自動提醒「該安排升級突破了」
   - 例：言情類提醒「感情線已經5章沒推進」

### 階段三：寫作後審閱系統

**點評維度升級（從11項擴展到15項）：**

原有11項保留，新增：
12. **類型契合度** — 是否符合該類型讀者的核心期待
13. **伏筆健康度** — 已埋伏筆的回收狀態
14. **角色弧線進度** — 主角成長是否按計劃推進
15. **商業節奏評估** — 付費節點前後的內容質量

### 階段四：數據化創作儀表板

**作者可視化面板：**
```
┌─────────────────────────────────────────┐
│  爽點密度圖    [▓▓▓░░▓▓▓▓░░░▓▓▓▓▓]     │
│  角色出場統計  主角:85% 女主:40% 反派:25% │
│  伏筆狀態     已埋:12  已收:7  待收:5    │
│  節奏曲線     [鋪墊→衝突→高潮→緩和→...]  │
│  字數統計     本卷:15萬  總計:45萬        │
└─────────────────────────────────────────┘
```

## 六、各類型速查表（AI 生成時快速參考）

### 玄幻類速查
- 必備元素：升級體系、金手指、打臉、地圖升級
- 爽點頻率：每3章一小爽，每15章一大爽
- 忌諱：升級太快無挑戰、反派智商下線、設定崩壞

### 仙俠類速查
- 必備元素：修煉體系、宗門政治、天劫、悟道
- 爽點頻率：每5章一小爽，每20章一大爽（節奏偏慢）
- 忌諱：修煉過程流水帳、境界劃分混亂

### 都市類速查
- 必備元素：系統/異能、日常反差、幽默感、社會場景打臉
- 爽點頻率：每2章一小爽，每10章一大爽（節奏最快）
- 忌諱：脫離現實太遠、後宮過多、劇情重複

### 歷史類速查
- 必備元素：歷史細節、權謀博弈、現代知識應用、家國情懷
- 爽點頻率：每5-8章一小爽，每25章一大爽
- 忌諱：歷史硬傷、主角過於開掛、配角臉譜化

### 言情類速查
- 必備元素：感情線推進、女主成長、宅鬥/職場衝突、甜寵場景
- 爽點頻率：每3章一甜/一虐，交替進行
- 忌諱：男主人設崩塌、女主聖母、感情線拖沓
